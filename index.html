Modelo longitudinal del vehículo

¿Qué fuerzas aceleran o frenan al camión?

Diagrama de cuerpo libre (pendiente)

Ecuación (Newton)

m · dv/dt = F_trac − F_roll − F_drag − m·g·sin(α)

El control longitudinal decide F_trac para compensar pendiente y resistencias.

Primero entendemos las fuerzas → luego diseñamos el control.

Control y modelo

Control lateral (Stanley)

Modelo cinemático (bicycle)
ψ̇ = (v / L) · tan(δ)
ẋ = v·cos(ψ)    ẏ = v·sin(ψ)
Velocidad v (cambia durante la maniobra)
Ángulo de dirección δ (giro/rumbo)

La geometría conecta la dirección δ con el movimiento.

Longitudinal controla fuerza · Lateral controla dirección.

Del modelo al software: control en tiempo discreto

1) Modelo discreto (Euler)

La ECU ejecuta el control cada Δt (ej. 20 ms):

\[ v_{k+1}=v_k+\Delta t\,a_k \] \[ \psi_{k+1}=\psi_k+\Delta t\,\dot{\psi}_k \] \[ x_{k+1}=x_k+\Delta t\,v_k\cos(\psi_k) \] \[ y_{k+1}=y_k+\Delta t\,v_k\sin(\psi_k) \]

2) Implementación en software

// Ciclo de control ECU
while (ECU_ON) {
  // Sensores
  x, y, psi, v = read_state();

  // Control lateral
  delta = theta_e + atan(k * e / v);

  // Dinámica (Euler)
  psi = psi + dt * (v/L) * tan(delta);
  x   = x   + dt * v * cos(psi);
  y   = y   + dt * v * sin(psi);

  // Actuadores
  send_steering(delta);
}

3) Tiempo real

  • Periodo fijo: Δt = 10–50 ms
  • Ejecuta en tarea determinística
  • Lectura → control → actuador
  • Errores se acumulan si Δt es grande
  • Base de simulación y ECU real
El vehículo no “resuelve ecuaciones”: itera estados en tiempo real.

Software en el vehículo

Sensores → ECU (software) → Actuadores (frenos / volante)

El software transforma “datos” en “acciones” en tiempo real.

¿Qué hace realmente el software?

El software es el puente entre la teoría y el vehículo real

El software se encarga de:

  • Leer sensores reales
  • Filtrar ruido y errores
  • Ejecutar ecuaciones de control
  • Tomar decisiones cada milisegundo
  • Enviar comandos seguros

En la práctica:

  • Código en C / C++ / Python
  • Sistemas en tiempo real
  • Simulación (SIL)
  • Pruebas con hardware (HIL)
  • Validación en el vehículo
Sin software, el control no existe.
